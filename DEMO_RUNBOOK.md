# BDC Agent Coaching — Demo Runbook

## Target Audience
- Stakeholders / product team
- BDC managers at automotive dealerships

## Demo Flow (10-15 min)

### 1. Dashboard Overview (2 min)
**Route:** `/`

**Talking Points:**
- "This is the BDC command center — all metrics refresh automatically via Dynamic Tables"
- Show KPI cards: total calls, conversion rate, avg sentiment, avg call score
- Toggle the dealer filter to show per-dealership views
- "No ETL scheduler needed — Snowflake Dynamic Tables handle the refresh pipeline"

**Snowflake Feature:** Dynamic Tables with auto-refresh

### 2. Agent Leaderboard (2 min)
**Route:** `/leaderboard`

**Talking Points:**
- "Agents are ranked by a composite score: 40% conversion, 30% call quality, 30% sentiment"
- Show how leaderboard updates per dealer
- "This data is served by Interactive Tables — sub-second latency, no warehouse spin-up"
- Click an agent name to navigate to their detail page

**Snowflake Features:** Interactive Tables, Interactive Warehouse

### 3. Agent Detail + AI Coaching (4 min)
**Route:** `/agents/{id}`

**Talking Points:**
- Show the agent's call history and scores
- **AI Summary:** "Watch — we're calling a Cortex Agent right now to generate a coaching summary"
  - Point out the metadata pill: model name, token count, cache hits, latency
  - "This uses claude-4-sonnet via Cortex Agent — no API keys needed, runs inside Snowflake"
- **Chat:** Ask a coaching question like:
  - "What are this agent's top 3 strengths based on their calls?"
  - "How does their objection handling compare to other agents?"
  - Point out the streaming response (SSE) and metadata after each message
  - "The agent is searching 15K call transcripts via Cortex Search to ground its answers"

**Snowflake Features:** Cortex Agent, Cortex Search, SSE streaming

### 4. Call Detail + Transcript (2 min)
**Route:** `/calls/{id}`

**Talking Points:**
- Show a single call with scores, sentiment, AI-generated summary
- Show the full transcript
- "All enrichments (sentiment, summary, disposition) were generated by Cortex AI functions"

**Snowflake Features:** Cortex AI Functions (pre-computed enrichments)

### 5. Data Pipeline (1 min)
**Route:** `/pipeline`

**Talking Points:**
- Show the pipeline flow visualization
- "This shows the data journey: Postgres → pg_lake → Iceberg → Snowflake → Dynamic Tables → App"
- "Everything is serverless — no infrastructure to manage"

**Snowflake Features:** Iceberg tables, Dynamic Tables, pg_lake

## Backup Plan

If the SPCS service is down:
```bash
# Check status
snow spcs service status BDC_COACHING_SERVICE -c aws_spcs --database BDC_DEMO --schema SPCS

# Check logs
snow spcs service logs BDC_COACHING_SERVICE -c aws_spcs --container-name backend --database BDC_DEMO --schema SPCS

# Redeploy (preserves URL)
bash deploy/spcs/deploy-spcs.sh --images
```

If Cortex Agent is slow or erroring:
- Summary and chat both have error handling — the UI will show an error message
- Check if the Cortex Search Service is healthy: `DESCRIBE CORTEX SEARCH SERVICE BDC_DEMO.COACHING.CALL_TRANSCRIPT_SEARCH`
- Verify the Agent exists: `SHOW AGENTS IN SCHEMA BDC_DEMO.COACHING`

## Pre-Demo Checklist

1. Open the dashboard URL in Chrome (not Safari — SSE works better)
2. Run `bash test-checklist.sh` to verify all endpoints
3. Have a Snowsight tab open to BDC_DEMO for showing objects
4. Clear browser cache if you see stale data
5. Make sure BDC_POOL compute pool is running: `DESCRIBE COMPUTE POOL BDC_POOL`
